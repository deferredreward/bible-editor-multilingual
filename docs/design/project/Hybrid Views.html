Hybrid views — context + control

Sketches exploring how to combine gatewayEdit's "all resources in one glance" with tcCreate's "scroll through verses for context" — while keeping a clearly active verse/note so edits never lose their place. Each option treats scroll, active state, and snap-back a little differently. Toggle through and let me know which threads are worth pulling.

Active verse / note (where you're editing) Context (scrollable, dimmed) Highlight on quote 📍 = snap-back-to-active

01 — Spine + Detail

A tall left "spine" lists every verse in the chapter as a one-line strip — read it like a table of contents. Center is the active verse with all 3 scripture columns + the notes/words/questions cards (gatewayEdit-style). Right rail keeps tA / tW reference articles pinned.

scrollspine + right rail scroll independently. Center pinned to active.

  • Whole chapter visible at all times → strong context
  • Center never moves → never lose your edit
  • Reference articles always at hand
  • Only 1 verse worth of resources visible
  • Spine items must be terse

02 — Freeze Row

The top of the screen is your frozen active row — full gatewayEdit-style cards for the current verse, never moves. Below it, a dense tcCreate-style stream of every other verse (compact). Click any row to promote it to the frozen position. Multi-select pins to compare.

scrollonly the bottom stream scrolls. Frozen row stays put.

  • Direct parallel to mental model: "what I'm on" vs "everything else"
  • Scrolling never loses edit position
  • Easy to "pin" a comparison verse
  • Vertical real-estate cost: frozen row takes ~40%
  • Only one full-detail verse at a time

04 — Focus Halo

Pure gatewayEdit grid in the middle, surrounded by a glowing halo so the active verse is unmistakable. Above and below sit thin "peek strips" of the previous and next verse with the same panel layout — same column for ULT, same for notes, etc., so your eye can scan vertically. Scrolling the whole grid shifts which verse is the halo.

scrollpage scrolls top→bottom of book. Halo follows the verse closest to viewport center (or whichever you click).

  • Familiar gatewayEdit ergonomics, with context
  • Column alignment makes A/B comparison effortless
  • Halo is unambiguous active indicator
  • Halo brightness must be tuned — could fight content
  • Peek strips are necessarily abbreviated

05 — Locked Columns

Every resource is its own long, vertical column listing every verse in the chapter (scripture, notes, words, questions). Each column has a "🔒 lock to active" toggle. Locked columns auto-scroll to the active verse; unlocked columns can be scrolled freely (e.g. flip ahead 5 verses in notes while keeping scripture put). A blue sync-bar across all columns shows where the active verse would be.

scrolleach column independent. Per-column lock keeps them aligned when wanted.

  • Massive context — see 10+ verses of every resource
  • Power-user mode: scroll notes ahead without losing scripture
  • Mirrors gatewayEdit panels but each one infinite-scroll
  • Lots of vertical scanning
  • Locking model needs onboarding

💡
Shared patterns across all 5:
  • One 📍 snap-back control per scrollable region — never lose the active verse.
  • Active state is always blue-edged and stays consistent across resource panels.
  • "Hover-peek" prev/next is implied — hovering a context strip can expand it in place without losing focus.
  • Keyboard: J/K or ↑/↓ = move active verse · G = snap-back · [ ] = move active note within verse.
