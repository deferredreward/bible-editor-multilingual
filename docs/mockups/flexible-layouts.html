Metaphor

A metaphor is a figure of speech in which someone speaks of one thing as if it were a different thing, because he wants people to think about how the two things are alike. Here the golden lampstand stands for the restored temple and God’s presence with his people.

  • If the metaphor would be understood by your readers, consider keeping it. If not, state the meaning plainly — e.g. “the temple, where God is present.”

Lampstand

Definition

A lampstand is a stand made to hold one or more oil lamps so their light can spread through a room. In the tabernacle and temple, the golden lampstand held seven lamps and stood in the Holy Place.

Translation Suggestions

  • This term could be translated as “stand for a lamp” or “structure that holds a lamp.”
  • For the temple lampstand, consider “lampstand of seven lamps.”

(see: gold, olive tree)

Definition

A lampstand is a stand made to hold one or more oil lamps so their light can spread through a room. In the tabernacle and temple, the golden lampstand held seven lamps and stood in the Holy Place.

Translation Suggestions

  • This term could be translated as “stand for a lamp” or “structure that holds a lamp.”

Layout schema (LayoutSpecV1)

Each preset (and each user-saved layout) serializes to a LayoutSpecV1. Region size values are flex fractions that sum to 1; the rail is outside the flex regions. versions: "inherit" means “use the project’s configured version set.”

TypeScript shape

interface LayoutSpecV1 {
  v: 1;
  id: string;
  name: string;
  builtin: boolean;
  requires?: "translation";
  orientation?: "horizontal" | "vertical";   // default "horizontal"
  rail: { visible: boolean };
  regions: { id: string; size: number; host: HostV1 }[];
}

type HostV1 =
  | { kind: "scripture"; mode: "stacked" | "columns" | "book";
      versions: string[] | "inherit" }
  | { kind: "resources"; tabs: ("notes" | "words" | "questions")[];
      initialTab: string; allowAlignment: boolean; allowSearch: boolean;
      notePairAxis?: "horizontal" | "vertical";    // default "horizontal"
      display?: "tabs" | "stacked";                // default "tabs"
      showAssociatedArticles?: boolean }           // default false
  // PROVISIONAL — pending review:
  | { kind: "article"; resourceType: "tw" | "ta";
      pairAxis: "horizontal" | "vertical"; showOccurrences: boolean };

Classic



      
Translate Notes

      
Vertical orientation stacks scripture over notes; the resources host carries notePairAxis for the per-note source/target axis.

      


      
Translate Words

      
The article host kind is provisional pending review. The rail expresses the article nav list; a single region holds the source/target article editor.

      


      
Book Package Review

      
The resources host uses display: "stacked" (all panels visible in a scroll stack, no tabs) and showAssociatedArticles: true (tA/tW articles linked from the active note/term appear adjoined under their list). Per-panel minimize/expand is runtime state stored in layout overrides, not part of the layout definition.

      


      
Evolved model — movable panels

      
To let users move panels between columns and add panes like an original-language view, a region evolves from hosting a single host to holding an ordered, movable list of panel instances. Regions stay the slots; panels become the movable units — still column/slot-based, not free-floating windows. Every preset above will be re-expressed in this model (a scripture host becomes a scripture panel; a resources host becomes its individual notes/words/questions panels).

      
interface RegionV2 { id: string; size: number; display?: "stacked" | "tabs"; panels: PanelInstance[] }
interface PanelInstance {
  id: string;                    // instance id, e.g. "scripture-1"
  type: "scripture" | "original" | "notes" | "words" | "questions"
      | "taArticle" | "twArticle" | "alignment" | "search";
  minimized?: boolean;
  size?: number;                 // optional within-region fraction
  config?: Record<string, unknown>;  // scripture:{mode,versions} · resources:{pairAxis} · original:{resource:"uhb"|"ugnt"}
}


      
Flexible

      
Two columns, each a movable panel stack. Panel moves/reorders and per-panel minimize are runtime state stored in layout overrides.

      


      
Two layers — containers & pane arrangement

      
A layout has two separable parts. (1) Container — a tree of horizontal/vertical splits that defines the frame: columns, a top section, or nested combinations. (2) Pane arrangement — which panels sit in each region, their order, and each panel's own config (scripture mode + versions, note source/target axis, tabs vs stacked, minimize). A preset bundles a container frame + pane placement + pane configs; a user can change either layer and save. Translate Notes' “top section” is simply a vertical split — reusable inside any layout, including Flexible.

      
type Node = SplitNode | PanelRegion;
interface SplitNode   { kind: "split"; orientation: "horizontal" | "vertical"; size?: number; children: Node[] }
interface PanelRegion { kind: "region"; size?: number; display?: "stacked" | "tabs"; panels: PanelInstance[] }
interface LayoutSpec  { v: 2; id: string; name: string; builtin: boolean; requires?: "translation";
                        rail: { visible: boolean }; root: Node }
